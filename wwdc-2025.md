## [Explore concurrency in SwiftUI](https://developer.apple.com/videos/play/wwdc2025/266)
- `@MainActor` isolated as default.
- From Swift 6.2. `Project Settings` → `Default Actor Isolation`에서 지정.
- SPM에서도 `.target(swiftSettings:)`를 이용해 모듈 별로 지정 가능.
- When a SwiftUI API requires you to write a sendable function, the framework will provide most of the variables you need as function arguments. (e.g. `Shape.path()`, `visualEffect()`, `Layout`, `onGeometryChange()`)
- SwiftUI's action callbacks are synchronous by design to ensure immediate UI updates.

## [Embracing Swift Concurrency](https://developer.apple.com/videos/play/wwdc2025/268)
- From Swift 6.2. `Project Settings` → `Default Actor Isolation`에서 지정. `MainActor` 또는 `nonisolated`.
- `URLSession`의 async 함수는 background 에서 돌아가게 되어 있지만... 우리 함수는 명시적으로 `@concurrent` 표시해야함.
- `Sendable`을 conform하고 있으니, 안전하게 actor를 오갈 수 있음.
- 이건 좀 헷갈리네. `@MainActor`로 지정된 함수 호출을 `@concurrent` 함수 내에서 하면, 호출 이후는 main actor로 변경되는 것인가?
- Actor objects aren't tied to a single thread like the main actor is.
